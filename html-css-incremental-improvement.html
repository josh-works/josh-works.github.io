A Little CSS Goes a Long Way

by Josh Thompson

heavily inspired by and copied from Web Design in 4 Min (jgthms.com)

I wanted to learn some basic styling. I've done back-end development most of my professional career, hardly ever had to weigh in on the front-end.

But, over here on my own personal website, I've got a bunch of modifications I want to make.

Specifically, I want come up with a way to add footnotes and annotations to my own writing. Maybe something similar to how Nate Berkopec (example) or Tim Urban (example) does it.

So, I'm here to talk about HTML, CSS, and JavaScript, but what is more important than all of those things?

Words, Ideas, and Memes

Certainly many websites deliver things besides words (Netflix, Youtube) but I'd argue most of stuff we do on the web comes down to reading text.

For example, you're reading this text right now

Let's improve the readability a little bit, by limiting the line length.

in our styles.css document:

    body {
      margin: 0 auto;
      max-width: 50em;
    }

(If you're on mobile, you won't see a difference)

It's important that words be easily read, and easily understood, as far as is possible.

These words are sometimes called "content", but I would argue that labeling this as simple 'content' undervalues what is happening here.

See, people are memetic.

I'm broadly defining "people are memetic" as people are strongly influenced by ideas and beliefs, and ideas and beliefs can spread through a person's mind and a group of people just like a virus might.

But the best words, the most infectious ideas, if presented in an unreadable way, will fail to have an effect.

Anyway, we ought to care about legibility.

Besides limiting the line length, we ought to style the text itself.

Lose the serifs

I prefer a font without serifs, or "san-serif". Lets add to our styles.css the following: 
    body: {
      font-family: "Helvetica", "Arial", sans-serif;
    }

But now, the text feels a little crowded, don't you think? Especially on long lines where the words wrap without line breaks. This paragraph is getting to be rather long, and it's feeling crowded already. If I had the audacity to make this paragraph multiple (long) sentances long, the way it would wrap would make it more than just a daunting block of text to wade into -- it would make it hard for the eye to follow along and avoid skipping up and down.

What to you need beyond a better font?

Text Spacing

By adding a little whitespace here and there, the text can "breath". See how the long lines wrap with a little more space between them? It helps the eye follow along on the current line instead of jumping above or below.

I know some die-hard readers insist that the text should be dense, and the form factor doesn't matter, but this is my website, so I can do whatever I want. ðŸ˜„

  body {
    line-height: 1.5;
    padding: 4em 1em;
  }
  
  h2 {
    margin-top: 1em;
    padding-top: 1em;
  }

Now that we have spacing sorted, what might help present information in a more compelling and easily-digested way? That's right. Modify the color and the contrast!.

Color and Contrast

Now code blocks are set apart visually from the rest of the page.

    code,
    pre {
      background: #eee;
    }

    code {
      padding: 2px 4px;
      vertical-align: text-bottom;
    }

    pre {
      padding: 1em;
    }
Lets style the links

Style the Links

The default link color isn't great, and it varies based on if you've clicked it before. Doesn't look great, so lets standardize our links on something that looks a bit nicer:

Lets upgrade the links:

  a {
    color: #268bd2;
    text-decoration: none;
  }

  a:hover,
  a:focus {
    text-decoration: underline;
  }

Next lets upgrade our code blocks.

Upgrade the code blocks

The code blocks are still kinda bland. We can add subtle accents to the left side and bottom:

    html.step6 code,
    html.step6 pre {
      background: #f5f7f9;
      border-bottom: 1px solid #d8dee9;
      color: #a7adba;
    }

    html.step6 pre {
      border-left: 2px solid #69c;
    }
